> **Codex Remote Web**
>
> 用浏览器远程查看和控制自己电脑上的 Codex，会话仍然完全由 Codex 管理。

这样复杂度最低，而且后面 Codex 升级基本不受影响。

---

# 一、最终架构

```text
┌─────────────────────┐
│ 手机浏览器/电脑浏览器 │
└─────────┬───────────┘
          │ HTTPS/WSS
          ▼
┌─────────────────────┐
│ Cloudflare Worker   │
│ + Durable Object    │
└─────────┬───────────┘
          │ WSS
          ▼
┌─────────────────────┐
│ Windows Agent       │
│ (Go 单文件程序)      │
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│ Codex CLI/Desktop   │
└─────────────────────┘
```

---

# 二、核心原则

不要：

❌ 存聊天记录

❌ 自己实现会话系统

❌ 自己实现 AI

❌ 自己管理上下文

全部交给 Codex。

你只负责：

```text
查看会话
查看状态
发送消息
实时输出
审批操作
```

---

# 三、Worker职责

Worker只做：

### API

```http
GET  /api/sessions
GET  /api/session/{id}
POST /api/send
GET  /api/status
```

### WebSocket

```text
/ws/client
/ws/agent
```

### 用户认证

推荐：

[Cloudflare Access](https://developers.cloudflare.com/cloudflare-one/policies/access/?utm_source=chatgpt.com)

直接使用：

* Google
* GitHub
* OpenAI

登录。

无需自己写 JWT。

---

# 四、Durable Object

负责：

```typescript
class CodexRoom {
    agentSocket
    clients[]
}
```

实现：

```text
消息转发
状态同步
在线检测
```

例如：

```json
{
  "type":"stream",
  "session":"abc",
  "content":"正在分析..."
}
```

直接广播给所有网页。

# 五、Windows Agent

建议 Go。

启动：

```bash
codex-agent.exe
```

常驻后台。

---

## Agent维护状态

```go
type State struct {
    CurrentSession string
    Running bool
    Queue []Message
}
```

---

## Agent连接Worker

```go
wss://codex.xxx.com/ws/agent
```

保持长连接。

---

## Agent发现会话

优先使用：

```bash
codex resume --list
```

如果没有对应命令：

读取：

```text
~/.codex/
```

中的会话数据。

---

返回：

```json
[
  {
    "id":"abc",
    "title":"修复登录"
  },
  {
    "id":"def",
    "title":"重构支付模块"
  }
]
```

---

# 六、如何获取历史记录

有两个方案。

## 方案A（推荐）

Agent直接读取 Codex 会话文件。

```text
~/.codex/sessions
```

解析后返回。

优点：

* 快
* 不影响运行中的会话

---

## 方案B

执行：

```bash
codex export
```

获取完整历史。

适合后续做：

```text
搜索
导出
分享
```

# 七、实时输出

这是最重要的部分。

Agent启动Codex：

```go
cmd := exec.Command(...)
```

监听：

```go
stdout
stderr
```

每收到内容：

```json
{
  "type":"stream",
  "session":"abc",
  "content":"正在分析 LoginService..."
}
```

立即推送。

网页实时显示：

```text
正在分析 LoginService...

发现空指针异常...

开始修复...
```

效果类似 ChatGPT 流式输出。

# 九、防止消息冲突

这是你的重点需求。

---

## 状态机

```text
IDLE
 ↓
RUNNING
 ↓
WAITING
 ↓
IDLE
```

---

Agent：

```go
if running {
    reject()
}
```

或者：

```go
enqueue()
```

---

推荐排队。

例如：

当前：

```text
修复登录Bug
```

正在执行。

手机发：

```text
增加日志
```

加入：

```json
[
  "增加日志"
]
```

当前任务结束：

自动发送。

# 十、电脑和手机同时操作

必须避免。

Agent维护：

```json
{
  "owner":"web"
}
```

或者：

```json
{
  "owner":"desktop"
}
```

---

如果网页正在控制：

```text
🔒 当前由 Web 控制
```

电脑输入框禁用（如果你后续能接管终端）。

---

更简单：

只允许一个输入源。

谁先发送消息：

```text
获得控制权
```

5分钟无操作释放。

类似 SSH Session。

# 十一、审批功能

Codex经常出现：

```text
Allow command?

[y] Yes
[n] No
```

Agent识别：

```json
{
  "type":"approval",
  "command":"git push"
}
```

网页弹出：

```text
批准执行？

git push

[允许]
[拒绝]
```

点击：

```json
{
  "type":"approve"
}
```

Agent回写。

# 十五、开发顺序

第一阶段

* Agent连接Worker
* WebSocket通信
* 在线状态

第二阶段

* 获取会话列表
* 查看历史记录

第三阶段

* 发送消息
* 实时输出

第四阶段

* 排队机制
* 运行状态

第五阶段

* 审批窗口
* 登录鉴权

最终就是一个：

```text
https://codex.yourdomain.com
```

打开即可看到所有 Codex 会话、当前运行状态、实时输出，并且能从手机浏览器继续向同一个会话发送消息，而无需开发原生 App。
